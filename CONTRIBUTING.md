# Contributing to ChurchApps

Thanks for helping build free, open-source software for the Church! 🎉

## Where things live

- **Report a bug or request a feature** — open an issue on **[ChurchAppsSupport](https://github.com/ChurchApps/ChurchAppsSupport/issues)** for anything in the B1.church or Lessons.church family, or on **[FreeShow](https://github.com/ChurchApps/FreeShow/issues)** for FreeShow. Most of our app repos have issues turned off on purpose so reports stay in one place.
- **Ask a question / say hi** — join our [Slack](https://join.slack.com/t/livechurchsolutions/shared_invite/zt-i88etpo5-ZZhYsQwQLVclW12DKtVflg).

## Contributing code

1. Read the [developer setup guide](https://churchapps.org/dev).
2. Fork the relevant repo and create a branch.
3. Make your change. Keep it focused and match the surrounding code style.
4. Open a pull request describing what and why. Link the related ChurchAppsSupport issue if there is one.

Not sure where to start? Look for [`good first issue`](https://github.com/ChurchApps/ChurchAppsSupport/labels/good%20first%20issue) on ChurchAppsSupport, or ask in Slack.

## Ways to help that aren't code

Translations, documentation, testing, design, and spreading the word all matter. And if you'd like to keep this software free for churches, please consider [sponsoring us](https://github.com/sponsors/ChurchApps).
